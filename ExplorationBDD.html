Introduction¶

Ce projet explore divers aspects de la manipulation de données à l'aide de Python, en se concentrant sur les opérations de conversion, de dénormalisation, et de jointure de données. Mon projet débute par la transformation de données structurées en format CSV vers le format JSON, qui est plus flexible. Par la suite, le projet s'aventure dans la dénormalisation des données. Enfin, le projet utilise les bases de données Redis et MongoDB pour stocker et gérer ces données mais pourquoi utilise t'on mongodb et redis ?.

MongoDB, en tant que base de données orientée documents, brille par sa capacité à stocker des données flexibles et riches. Mongodb utilise des fichiers json qui permettent une représentation structurée des données, qui sera beaucoup plus adaptée à notre approche de dénormalisation. Ces documents peuvent contenir des informations variées . Ainsi, MongoDB s'affirme comme très bonne solution pour le stockage de données complexes.

D'un autre côté, Redis, en tant que base de données en mémoire à clé-valeur, excelle dans la rapidité d'accès aux données. Grâce à sa structure simple, Redis offre des performances très rapide pour les opérations de lecture et d'écriture. Dans notre projet, nous verrons comment Redis est un spécialiste dans la rapidité de traitement des données .

PARTIE 1 : Travail sur le format JSON¶

Conversion CSV en JSON¶

Objectif et Importance¶

Cette section du projet se concentre sur la conversion de données du format CSV au format JSON . Cette étape est cruciale pour plusieurs raisons :

Facilité de Manipulation¶

Le format JSON est largement utilisé pour sa facilité de manipulation, en particulier dans les langages de programmation comme Python. Les structures de données en JSON sont flexibles.

Interopérabilité et Stockage¶

Le JSON est également un format standard pour l'échange de données sur le web, Par exemple, lors de l'utilisation d'API pour interagir avec les données, les requêtes génèrent des résultats formatés en JSON. De plus, il est plus adapté au stockage dans des systèmes de bases de données NoSQL tels que MongoDB.

Script csv_to_json¶

Ce script utilise la bibliothèque pandas pour convertir un fichier CSV en fichier JSON. Il lit le fichier CSV spécifié, traite les colonnes pour les dates si nécessaire, et sauvegarde les données dans un fichier JSON avec une orientation des enregistrements, ce qui signifie que chaque ligne du CSV deviendra un objet JSON distinct dans un tableau JSON.

import pandas as pd

def convert_csv_to_json(input_csv_path, output_json_path, target_columns, date_columns=None):
    # Charger les données depuis le fichier CSV en utilisant une tabulation comme délimiteur
    data_frame = pd.read_csv(input_csv_path, delimiter='\t', header=None, names=["RawData"])
    
    # Diviser la colonne "RawData" en plusieurs colonnes en utilisant la tabulation comme séparateur
    parsed_data = data_frame["RawData"].str.split('\t', expand=True)
    
    # Renommer les colonnes avec les noms spécifiés
    parsed_data.columns = target_columns

    # Si des colonnes de date sont spécifiées, convertir les valeurs en format de date spécifié
    if date_columns:
        for date_col in date_columns:
            parsed_data[date_col] = pd.to_datetime(parsed_data[date_col], format='%d/%m/%y').dt.strftime('%d/%m/%y')

    # Exporter les données au format JSON avec une indentation de 2 espaces
    parsed_data.to_json(output_json_path, orient='records', indent=2)

Denormalisation des données¶

Le script Python prend deux fichiers JSON, un contenant des informations sur les clients et l'autre sur les réservations, puis dénormalise ces données en les combinant dans un fichier JSON dénormalisée. Pour chaque client, le script rassemble ses réservations associées, créant ainsi une structure où chaque client a une liste de réservations. Le résultat est enregistré dans un nouveau fichier JSON appelé denormalized_data.json.

Jointure des Données JSON¶

Objectif de la Jointure¶

Cette partie du projet implique la jointure de deux ensembles de données JSON, Avions.json et Vols.json, sur un attribut commun. Le script implémente une Jointure Interne (Inner Join) entre les ensembles de données Avions.json et Vols.json. Seuls les enregistrements ayant une correspondance exacte sur l'attribut NumAv dans les deux fichiers sont retenus.

Processus de Jointure¶

Chaque paire d'entrées des deux fichiers est comparée. Si les valeurs de l'attribut spécifié (NumAv dans ce cas) correspondent, les informations des deux entrées sont combinées en un seul objet.

Avantages de cette Approche¶

  • Données Consolidées : Permet de consolidées les donneés , ce qui permets de trouver les données a un endroit consolidées.
  • Flexibilité et Réutilisation : La fonction de jointure peut être appliquée à différents ensembles de données et attributs, et sera réutilisée pour intégrer les données dans MongoDB.

Résultat¶

Le script produit un ensemble de données JSON jointes, où chaque objet représente une combinaison des données de vol et d'avion basée sur le numéro de vol.

import redis
import time

def publish_messages(channel):
    r = redis.Redis()
    message = "SALUT"

    while True:
        r.publish(channel, message)
        print(f"Message publié : {message}")
        time.sleep(3)  # Attendre 3 secondes

channel_name = "mon_canal"
publish_messages(channel_name)

import redis

def subscribe_to_channel(channel):
    r = redis.Redis()
    pubsub = r.pubsub()
    pubsub.subscribe(channel)

    print(f"Abonné au canal : {channel}")
    for message in pubsub.listen():
        if message['type'] == 'message':
            print(f"Message reçu : {message['data'].decode()}")

Introduction aux Index dans MongoDB¶

Dans MongoDB, les index jouent un rôle crucial dans l'optimisation des performances des requêtes. Un index dans MongoDB est une structure de données spéciale qui améliore la vitesse des opérations de recherche au sein des collections. Lorsqu'un index est créé sur un champ, MongoDB utilise cet index pour accéder rapidement aux données sans avoir à parcourir l'ensemble de la collection. Cela est particulièrement utile pour les grandes bases de données, où les recherches sans index peuvent devenir lentes et coûteuses en ressources.

import csv
import time
from pymongo import MongoClient

def perf_mongo_no_index(csv_file, n):
    myclient = MongoClient()
    mydb = myclient["mydatabase"]
    mycol = mydb["mycollection"]

    # Vider la collection
    mycol.drop()

    with open(csv_file, encoding='utf-8') as csvfile:
        my_reader = csv.DictReader(csvfile, delimiter='\t')
        my_data = [my_row for my_row in my_reader]

        print('Début de l\'évaluation de performance sur', n, 'entrées')

        # Temps de départ
        st = time.process_time()

        for my_row in my_data[:n]:
            # Test si la clé existe et remplace
            mycol.replace_one({my_row['M']: 1}, {my_row['M']: 1}, upsert=True)

        # Temps de fin
        et = time.process_time()

        # Calculer le temps d'exécution
        res = et - st
        print('Temps d\'exécution CPU:', res, 'secondes')

        # Compter les doublons
        duplicates = n - mycol.count_documents({})
        print('Doublons trouvés:', duplicates)

        return res

Conclusion : Redis, MongoDB et JSON dans la Gestion de Données¶

Redis : Rapide et Réactif¶

Redis est un système de base de données qui travaille directement avec la mémoire de l'ordinateur, ce qui le rend super rapide. Il est parfait pour des tâches qui nécessitent une réponse immédiate, comme les chats en ligne ou les jeux. Redis est aussi vraiment bon pour gérer des données qui changent souvent, grâce à ses outils simples mais puissants.

MongoDB : Flexible et Facile à Utiliser¶

MongoDB est un type différent de base de données qui stocke les informations un peu comme des documents JSON, ce qui le rend très pratique pour travailler avec des données variées. Il n'a pas besoin de structures rigides, donc on peut facilement ajouter ou changer des informations. C'est super pour des gros projets où les données peuvent être différentes et doivent être accessibles rapidement.

JSON : Le Langage Commun¶

JSON est une façon simple d'écrire des données qui peut être lue par beaucoup de langages de programmation. C'est comme un langage commun qui aide les ordinateurs à se comprendre, surtout quand ils partagent des informations sur internet. C'est pratique parce que c'est facile à lire et à écrire, autant pour les humains que pour les machines.

En Résumé¶

En utilisant Redis, MongoDB et JSON ensemble, on crée une super équipe pour gérer des données. Redis apporte la vitesse, MongoDB offre la flexibilité, et JSON fait en sorte que tout soit facile à utiliser et à partager. C'est un mélange parfait pour des applications modernes qui ont besoin de gérer beaucoup d'informations de manière efficace et rapide.
